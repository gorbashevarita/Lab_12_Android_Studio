## Вопросы:
### Какие три метода вызываются при запуске приложения?
onCreate(), onStart(), onResume()
### Чем отличается onPause() от onStop()?
onPause() вызывается, когда активность теряет фокус,

onStop() вызывается, когда активность полностью невидима
### Когда вызывается onRestart()?
onRestart() вызывается, когда активность переходит из состояния Stopped в Started
### Что происходит при повороте экрана?
Поворот экрана вызывает configuration change (изменение конфигурации)

Android полностью уничтожает Activity (onDestroy)

Затем создаёт заново (onCreate)

Все данные теряются
### В чём разница между remember и rememberSaveable? 
**remember:**

1.Cохраняет значение только при рекомпозиции

2.Теряет значение при пересоздании Activity
remember

**Saveable:**
1. Сохраняет значение при рекомпозиции

2. И при configuration changes (поворот, смена языка и т.д.)

### Почему onCreate() должен вызывать super.onCreate()?

выполняет инициализацию родительской реализации Activity (например, настраивает контекст, тему, жизненный цикл);

без него активность не будет корректно работать — могут не загрузиться ресурсы, не инициализироваться компоненты;

порядок вызова важен: super.onCreate() должен идти перед настройкой UI и другими операциями в onCreate().

### Для чего используется класс Log?
1. вывода отладочных сообщений в лог-каталог (LogCat) Android Studio;
2. диагностики и отладки приложения — позволяет отслеживать выполнение кода, значения переменных, ошибки;
3. логирования событий, исключений, состояний компонентов;
4. содержит методы с разными уровнями логирования: Log.d() (debug), Log.e() (error), Log.i() (info), Log.w() (warning) и др.

### Какой метод вызывается только один раз за время жизни Activity?

Метод onCreate() вызывается только один раз за время жизни Activity — при её создании.
